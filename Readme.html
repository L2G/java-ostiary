Running as an application

The Java client will run as either an application or as an applet. To run it as an application, the following command should work: 

   java -jar ostclient.jar

Running as an Applet

The file AppletExample.html shows the HTML code used to embed the applet into a web page. There are two optional parameters that can be provided to the applet:

  • OSTHOST - The host the applet should connect to.
  • OSTPORT - The port to connect on at the above host.

If they are not provided, the associated fields in the applet will be blank.

Note that applets, by default, only have permission to intiate socket connections back to the host they were served from. So (unless you configure your browser to give the applet extra permissions) it will only be able to connect to the webserver that served up the webpage it's embedded in. Which therefore implies that Ostiary will need to be running on that webserver.

Source code

The source for the Java client is in the "/src" directory. A simple script to build it called "make.sh" is provided.
